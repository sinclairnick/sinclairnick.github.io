About Me

Recently I graduated from Victoria University of Wellington with a degree in psychology and philosophy. This begs the question, why do I want to become a web developer? At university I took web development courses under the Commerce department, but despite getting top grades for my websites, I found myself learning more through online resources than the courses themselves. Thus, I decided to postpone my web dev education until I could switch my dedication from psychology and philosophy back to web development, after graduating.

As soon as I had graduated university, putting 100% into my final year, I began to self-teach JavaScript, HTML, CSS and others, picking up where I had left off a few years prior. At this stage, some of the technologies I am familiar with include:

Other than basic Java which I learned during University, Python was the first language I was competent using. I found Python to be more beginner--friendly than Java, as concepts such as type checking and scope declarations were not required. This reignited my passion for coding and allowed me to learn the primary principles of coding before focussing on more complex paradigms and techniques.

While Python was enjoyable to use, my interest lay in being able to make web apps and web sites, which meant JavaScript was the next stepping stone.

Like many, I initially found JavaScript to be more convoluted and illogical than, say, Python. However, as I gradually came to terms with the philosophy of the language, and became familiar with the inner workings of JavaScript, my confusion and frustration would later turn into appreciation for the nuances of this charismatic language.

At this point I would need to learn HTML and CSS to utilise JavaScript, the language of the web.

As mentioned earlier, I was experienced in HTML from my university days, so little was needed to get me back up and running with HTML. But as time progresses, I have noticed subtle improvements in my use of HTML elements and the organization of my web apps.

Like HTML, I was already familiar with CSS. I soon figured out that familiarity with CSS would not cut the mustard when projects become complex, and so my CSS skills have improved greatly, after properly utilizing the available tools like grid, flex box, animations etc., not to mention the frameworks like Bootstrap which do much of the heavy lifting.

After I felt comfortable turning website concepts into functional and good-looking (but ultimately static) web pages, I felt the need to delve into NodeJS to enter the back-end world of web development. Before even learning the Node methods and API, I made sure to understand the protocols and structure of the web. I had/have a veritable fascination with the inner workings of http communication, and the way Node integrates C++ and JS to bring JS to the server-side.

Electron provided a way to build a desktop app with the languages, libraries and frameworks I had already learned. I jumped into Electron shortly after Node, as a way to practice using Node, and to make something that was not possible purely within a browser.
This is what allowed me to create juke

To bring my front-end skills to an acceptable level I needed to learn a Front-end JavaScript framework. After initially understanding the basics of React, I looked at Vue.js for comparison, and found Vue to be adequate for my needs. I learned Vue as I was creating juke.

While I am familiar with Vue, I will resume learning React, as I am aware of its benefits.

Sass is the latest technology I have invested time into, and was used to make this web page. Sass (SCSS syntax) took a short time to learn, but has been a huge time-saver which I wish I had picked up earlier.
